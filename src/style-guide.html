Naming Conventions

General

CSS class names conform to the BEM (Block-Element-Modifier) naming convention. BEM's strict naming rules can be found here.

CSS class names for component, layout, and utility classes have the following prefixes.

Grid

The grid uses two main classes, .l-row and .l-col. The former indicates a row and the latter indicates a column.

To alter the width of a column, you must use a modifier. The syntax for column modifiers is l-col--{breakpoint}-{width}-of-{columns}, where {breakpoint} is an element in $ds-layout-grid-sizes, {width} is an integer between 1 and {columns}, and {columns} is the value of $ds-layout-grid-cols (the total amount of columns in the grid).

For example, suppose you set $ds-layout-grid-cols to 12, generating a 12-column grid. Here are some (not all) of the column modifiers that would be generated.

Column Modifier

Definition

l-col--xs-1-of-12

Column has a width of 1 / 12 = ~8.3% at breakpoint xs.

l-col--sm-3-of-12

Column has a width of 3 / 12 = 25% at breakpoint sm.

l-col--md-6-of-12

Column has a width of 6 / 12 = 50% at breakpoint md.

l-col--lg-12-of-12

Column has a width of 12 / 12 = 100% at breakpoint lg.

As you can see, to obtain the width of the column at a particular breakpoint, simply divide {width} by {column}.

Buttons

The following modifiers apply to all buttons:

  • a-btn--heavy makes the button text bold.
  • a-btn--small makes the button small.
  • a-btn--large makes the button large.
  • a-btn--outline makes the background transparent with a color outline.
  • a-btn--selected makes the button appear the same as if it were being hovered.

Default

Use the default button when no other color is suitable. The default button uses the a-btn--default modifier.

Primary

Use the primary button for facilitating the execution of innocuous actions (e.g., opening up a modal window). The primary button uses the a-btn--primary modifier.

Danger

Use the danger button to indicate that an action will result in the removal of information (e.g, unsubscribing from a Platform Event). The danger button uses the a-btn--danger modifier.

Success

Use the success button to indicate that an action will result in the addition of content (e.g, subscribing to a Platform Event). The success button uses the a-btn--success modifier.

Transparent

Use the transparent button when you do not want a colored button. The transparent button uses the a-btn--transparent modifier.

Disabled

All of the button variants can be disabled by simply setting the disabled attribute on the element to true.

Dropdown
